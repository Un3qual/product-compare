# Docs Guide

## Start Here

- `docs/NEXT.md`: active queue and default source of truth for "what should be done next?"
- `docs/plans/`: design and implementation docs for specific initiatives.
- `docs/decisions/`: locked decisions and scope posture.
- `docs/implementation-checklist.md`: historical delivery checkpoints, not the live queue.

## Maintenance Rules

- Keep `docs/NEXT.md` short, audited, and limited to currently actionable work.
- Put task-routing status in `docs/NEXT.md`; keep plan docs focused on scope, design, and execution details.
- When active work changes, update `docs/NEXT.md` and the referenced plan/design doc together.
- Remove completed initiatives from `docs/NEXT.md` instead of leaving a long tail of checked boxes in the active queue.
